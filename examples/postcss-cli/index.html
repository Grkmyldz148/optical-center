optical-center · postcss-cli

The no-bundler path. Pure PostCSS, no JavaScript at runtime, no Vite / webpack / Next.js. Icons come from real installed npm packages (lucide-static, heroicons, @fortawesome/fontawesome-free); the plugin resolves bare specifiers through Node's module resolution.

Each row below is the same SVG mounted twice. The geometric badge mounts the source SVG as-is — the icon's bounding-box centre lands on the crosshair. The optical badge adds one declaration — optical-center: auto — to the rule. At build time, the plugin rewrites every url('…svg') in that rule to an inline data URI whose viewBox is shifted by the icon's perceptual offset. The crosshair stays in the same place; the icon moves. 

/* what you write */                  /* what the plugin emits */
.icon {                              .icon {
  background-image: url('play.svg');   background-image: url("data:image/svg+xml,…");
  optical-center: auto;                --optical-center: auto;
}                                    }

Same icon, three different installed packages. The plugin resolves each bare specifier through node_modules and rewrites the url() identically — there is no package-aware logic, just Node module resolution plus SVG-shaped pixel analysis.

2 · background shorthand

The url() can live inside any declaration in the rule, not just longhand background-image. The plugin matches url('…svg') inside the shorthand and rewrites it in place.

lucide-static / heart.svg
background: var(--bg) url(…) center / 44px no-repeat;

3 · background-position independence

The perceptual shift is baked into the SVG's viewBox, not into the wrapper's positioning. Both badges below use background-position: 28% 50%; the optical badge stays at that position, with the icon's optical centre — not its bounding-box centre — landing there. The plugin does not fight your layout.

lucide-static / play.svg @ 28% 50%
Off-centre placement, perceptual centre still sits where you positioned it

4 · mask-image (re-tintable)

Combining mask-image: url('…svg') with a single background colour makes the icon re-tintable through CSS. The plugin treats the mask url() the same as a background url() — same rewrite, same shifted viewBox.

lucide-static / arrow-right.svg
mask + background: currentColor · icon shape recoloured by parent color

5 · mask shorthand

Same rewrite as mask-image, just authored as the mask shorthand. The url matcher cares only about what's inside the declaration value, not which property holds it.

lucide-static / chevron-right.svg
mask: url(…) center / contain no-repeat;

6 · content: url() on a pseudo-element

A ::before with content: url('…svg') mounts the SVG as a replaced element. The plugin walks the pseudo's rule too — there is nothing CSS-property-specific in the matcher.

lucide-static / bell.svg
.pseudo-bell::before { content: url(…); optical-center: auto; }

Why this example exists

To prove that anywhere PostCSS runs, optical-center can run. Tailwind, Next.js, Astro, Create React App, esbuild + postcss-loader, webpack + postcss-loader, standalone postcss-cli — same plugin, same one-line opt-in, same output. No runtime, no JS payload, no bundler-specific syntax.
