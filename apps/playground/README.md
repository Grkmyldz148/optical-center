# optical-center · playground

Interactive playground for the `optical-center` model. Two screens:

| Screen | What it's for |
|--------|---------------|
| **Playground** | Pick a curated icon, drag size + shape controls, see the geometric vs the pre-compiled optical viewBox side by side. Numeric breakdown of the offset. |
| **Stress test** | Every Iconify family (~227 collections) listed as click-to-expand sections. The optical toggle swaps each cell's `<svg viewBox=…>` between the source value and the corrected value. |

**Zero runtime cost, zero external API calls, zero npm scripts.**
The browser never runs the optical model. Icon bodies come from the
locally-vendored `@iconify/json` package, registered with
`@iconify/react`'s offline registry. Corrected viewBoxes are produced
on demand by `vite-plugin-optical-iconify` (Vite middleware backed by
`optical-center/node`) and cached on disk — the same function the
Babel and PostCSS plugins use in production builds. The browser just
picks one of two viewBox strings per cell.

## Run it

```bash
# from repo root
npm install
npm --workspace optical-center-example-playground run dev   # http://localhost:5173
```

The first time you click an unfamiliar family in the stress view, the
Vite plugin computes that family's optical offsets and writes them to
`node_modules/.cache/optical-iconify/<prefix>.json`. Subsequent loads
of the same family are instant. No manual precompute step.

## How it works

```
src/
  main.tsx                     # bootstrap + global CSS
  App.tsx                      # top-bar + view switch
  views/
    PlaygroundView.tsx         # curated 8-icon comparison stage
    StressView.tsx             # ~227 Iconify families · click-to-expand
  components/
    TopBar.tsx                 # brand + section tabs
    Badge.tsx                  # swaps `<svg viewBox>` on optical toggle
    OffsetReadout.tsx          # numeric breakdown of the viewBox shift
    IconifyFamily.tsx          # one family section + lazy data load
    IconifyCell.tsx            # one Iconify icon, viewBox-swapped on toggle
    PerfHud.tsx                # fps · frame · cells · fetch queue · loaded sets
    controls/                  # Slider · Segmented · Toggle · IconPicker · SwatchRow
  icons/
    icons.json                 # 8 curated icons (source of truth)
    offsets.generated.json     # ← auto-regenerated by the Vite plugin
    registry.ts                # merges the two into PlaygroundIcon[]
  lib/
    iconifyLocal.ts            # loader for /iconify/{bodies,offsets}/<prefix>.json
    familyLoadQueue.ts         # concurrency-limited manifest-load gate
    useCollections.ts          # one-shot /iconify/collections.json fetch
    usePerformance.ts          # rAF-driven FPS sampler
    theme.ts                   # HelmLab-generated CSS custom properties
    clsx.ts                    # tiny className helper
vite-plugin-optical-iconify.mjs  # serves Iconify data + lazy optical offsets
vite.config.ts
```

## The contract

For every icon shown on this page the runtime ends up with two
viewBox tuples:

```
geometric:   "0 0 24 24"                       (original, from @iconify/json)
optical:     "-1.0931 -0.6251 24 24"           (transformViewBoxFromSvg)
```

Toggling "optical correction" swaps the `<svg>` element's `viewBox`
attribute between them. No translate, no canvas, no model. The
package's own production paths (`optical-center/babel`,
`optical-center/postcss`) emit exactly the same corrected viewBox —
this playground just routes the same offline computation through a
Vite middleware instead of injecting it into JSX or CSS at build time.
